How It Works

01
Raw Notes → Obsidian Wiki
Internal data — case studies, transcripts, VA capabilities, reviews — gets structured into clean markdown files using Claude Code via terminal.
02
Wiki → GitHub Repo
The knowledge wiki is version-controlled and pushed to a GitHub repository via terminal. Every update is tracked. The brain compounds daily.

Repo → Claude Project Query Layer
The GitHub repo connects to a Claude Project. Anyone on your team can query the brain — before a call, during prep, or while writing copy.

Ad Copy Verification
Review this ad: "Hire a top-tier VA today. Experienced in all automation tools. Results guaranteed."
Issues flagged:

⚠ "All automation tools" — overly broad, can't be verified from data. Replace with specific tools: GHL, Zapier, Make, Notion.

⚠ "Results guaranteed" — no backing in case study data. Could attract complaints. Remove or add a specific, verifiable claim.

⚠ "Top-tier" — vague. Replace with: "Pre-vetted. Manager oversight. 24-hour replacement."

Stronger version: "Hire a GHL-certified VA this week. Pre-vetted, managed, replaced in 24hrs if it doesn't work."
